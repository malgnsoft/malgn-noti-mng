## 목적

테넌트 소속 **개별 사용자 계정**을 운영자가 관리한다. 가입 시 OTP/휴대폰/이메일 인증 실패 진단, 강제 비활성화, 임시 비밀번호 발급, 2FA 초기화 등 고객 지원 작업의 도구. CLAUDE.md §1.1.

## 주요 사용자

- `support` — **일** 단위 (고객 문의 응대 — 로그인 못 함, OTP 안 옴 등).
- `ops_admin` — 강제 비활성·임시 비번 등 mutating 액션.

## 주요 액션

- 사용자 검색 (이메일·휴대폰·이름·테넌트).
- 가입 인증(OTP/휴대폰/이메일) 실패 로그 확인.
- 임시 비밀번호 발급 (이메일 발송).
- 2FA(TOTP/passkey) 초기화(`Danger Zone`).
- 강제 비활성화 / 복구.
- 사용자가 속한 테넌트로 이동.

## 데이터 모델

### 사용자 (User)

```typescript
interface User {
  id: string
  email: string
  name: string
  phone: string                       // 암호화 저장
  tenantId: string
  role: 'admin' | 'manager' | 'user'
  
  status: 'active' | 'inactive' | 'pending_verification'
  statusReason?: string               // 비활성 사유
  
  twoFactorEnabled: boolean
  twoFactorMethod?: 'totp' | 'passkey'
  
  passwordChangedAt?: ISO8601
  lastLoginAt?: ISO8601
  lastLoginIp?: string
  lastLoginUa?: string
  
  createdAt: ISO8601
  updatedAt: ISO8601
}
```

### 인증 이벤트 (AuthEvent)

```typescript
interface AuthEvent {
  id: string
  userId: string
  type: 'otp' | 'email' | 'sms' | 'totp'
  success: boolean
  failureReason?: string              // 실패 사유 (코드 오류, 만료 등)
  attemptCount?: number               // 시도 횟수 (실패 시)
  ipAddress?: string
  userAgent?: string
  createdAt: ISO8601
}
```

### 로그인 이벤트 (LoginEvent)

```typescript
interface LoginEvent {
  id: string
  userId: string
  success: boolean
  ipAddress: string
  userAgent: string
  deviceType: 'web' | 'mobile' | 'desktop'
  failureReason?: string
  createdAt: ISO8601
}
```

## 검증 · 에러 처리

| 항목 | 검증 | 에러 메시지 |
| --- | --- | --- |
| **이메일** | 이메일 형식 + 고유성 | "유효한 이메일을 입력해주세요." |
| **이름** | 필수, 1~100자 | "이름을 입력해주세요." |
| **휴대폰** | 형식 (010/011 등) | "올바른 휴대폰 형식을 입력해주세요." |
| **임시 비밀번호** | 사유 필수 | "발급 사유를 입력해주세요." |
| **강제 비활성** | 사유 ≥10자 필수 | "비활성화 사유를 10자 이상 입력해주세요." |
| **2FA 초기화** | 확인 팝업 필수 | "정말 초기화하시겠습니까? (사용자는 재설정 필요)" |
| **세션 만료** | 확인 팝업 필수 | "모든 로그인 세션을 종료합니다. 계속하시겠습니까?" |

## 위험 액션 · 감사 로그

| 액션 | 위험도 | 감사 | 조건 |
| --- | --- | --- | --- |
| **임시 비밀번호 발급** | 중간 | ✓ 기록 | 사유 필수, 이메일 발송 기록 |
| **강제 비활성** | 높음 | ✓ 기록 | 사유 ≥10자 필수 |
| **강제 활성** | 중간 | ✓ 기록 | 승인자 확인 |
| **2FA 초기화** | 높음 | ✓ 기록 | 사유 필수, 사용자 재설정 필요 |
| **세션 전체 만료** | 높음 | ✓ 기록 | 사유 필수, 모든 기기 로그아웃 |
| **휴대폰 평문 조회** | 낮음 | ✓ 기록 | "한 번 보기" 버튼, 조회자 ID 기록 |
| **CSV 내보내기** | 낮음 | ✓ 기록(선택) | 내보낸 행 수 + 필터 기준 |

모든 위험 액션은 `AppRiskActionDialog` 사용:
- 위험도 표시 (높음/중간)
- 사유 입력 필드 (필수)
- 사용자 이메일 타이핑 확인 (강제 비활성/세션 만료)
- before/after 미리보기
