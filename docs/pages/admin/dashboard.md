> **라우트**: `/admin`
> **권한**: `super_admin` · `ops_admin` · `support` · `billing_admin` (전 운영자 공통 랜딩)

## 목적

운영자가 로그인 직후 가장 먼저 보는 화면. 서비스 전체 상태(활성 테넌트·금일 발송량·실패율·임박 한도·미답변 문의)를 한 화면에서 파악하고, 빠른 액션으로 상세 페이지로 이동한다. CLAUDE.md §1 전 도메인의 진입점.

## 주요 사용자

- `super_admin` · `ops_admin` — **일** 단위 (출근 직후 + 수시).
- `support` — 미답변 문의 카드 중심.
- `billing_admin` — 크레딧 소진 · 정산 카드 중심.

## 주요 액션

- 활성 테넌트 / 금일 발송 / 임박 한도 / 미답변 문의 KPI를 한 눈에 확인.
- 채널별 일별 발송 추이 차트로 비정상 급등·급락 감지.
- 실패율 차트에서 임계 초과 시 [실패율 분석](./report/failure.md)으로 드릴다운.
- 운영 알림 큐(인증서 만료 임박 · 한도 임박 · 미답변 문의)에서 카드 클릭으로 해당 상세 페이지 이동.
- 빠른 액션(테넌트 검색 · 발송 검색 · 점검 모드 토글)으로 자주 쓰는 작업 단축.

### 기간 선택

우측 상단 세그먼트 컨트롤:
```
[오늘] [7일] [30일]  (선택된 항목 강조)
```

### 1행: KPI 카드 (4개, col-span-3)

| # | 카드 | 표시 | 포맷 | 액션 |
| --- | --- | --- | --- | --- |
| 1 | **활성 테넌트** | 현재 활성 테넌트 수 | `1,234개` | [고객사 페이지로](./member/company.md) |
| 2 | **금일 발송** | 오늘 발송 건수 합계 | `567,890건` | [발송 조회로](./send/sms.md) |
| 3 | **임박 한도** | N일 내 소진될 테넌트 | `12개` (배지) | [한도 조정 필요](./member/company.md) |
| 4 | **미답변 문의** | 회신 대기 중인 1:1 문의 | `8건` (빨강) | [1:1 문의로](./ops/inquiry.md) |

각 카드는 **실시간 집계** (또는 1시간 주기) + 당일 대비 증감 % 표시.

### 2행: 차트 카드 (2개, col-span-6)

#### Chart 1: 채널별 발송 추이 (Line + Bar)

```
                    일별 발송량
                     ↑
          ╱╲         ╱
         ╱  ╲       ╱
        ╱    ╲     ╱
─────────────────────→ 일자 (선택 기간)

범례: [SMS] [RCS] [Kakao] [Email] [Push]
```

- X축: 선택 기간(1일~30일)의 각 일자
- Y축: 발송 건수 (로그 스케일 가능)
- 선: 채널별 색상 구분
- 마우스 hover: 일별 상세 수치 팝업

#### Chart 2: 실패율 추이 (Line)

```
                    채널별 실패율 (%)
                     ↑
                    3%─╱╲
                   1.5%╱  ╲  ╱
                    0%─────────→ 일자

범례: [SMS] [RCS] [Kakao] [Email] [Push]
```

- 임계값(예: 1%) 라인 점선으로 표시
- 초과 구간 강조 (배경색)

### 3행: 운영 알림 + 빠른 액션

#### 운영 알림 큐 (col-span-8)

카테고리별 알림 목록 (확장 가능):

```
┌─ 인증서 만료 임박 ───────────────┐
│ • FCM 인증서 (테넌트A)  14일 후  │
│ • APNs 인증서 (테넌트B) 7일 후  │
└──────────────────────────────────┘

┌─ 한도 임박 (월 한도 80% 이상) ──┐
│ • 테넌트C (SMS 월한도)  92%     │
│ • 테넌트D (Email)      87%     │
└──────────────────────────────────┘

┌─ 심사 대기 ──────────────────────┐
│ • 테넌트E (신청: 2일 전)         │
│ • 테넌트F (신청: 5일 전)         │
└──────────────────────────────────┘

┌─ 미답변 문의 ────────────────────┐
│ • [결제] 카드 인증 실패 (2시간)  │
│ • [기술] API 429 에러 (5시간)   │
└──────────────────────────────────┘
```

각 항목 클릭 → 해당 상세 페이지로 이동.

## 데이터 모델

### KPI 집계

```typescript
interface DashboardKpi {
  activeTenants: number              // 현재 활성 테넌트
  todaySendCount: number             // 오늘 발송 건수 (모든 채널)
  quotaAlertTenants: number          // N일 내 한도 임박 테넌트 수
  unansweredInquiries: number        // 미답변 1:1 문의 수
  
  // 변화도
  tenantDelta: number                // 오늘 신규 테넌트
  sendDelta: number                  // 어제 대비 발송량 변화율 (%)
}
```

### 시계열 데이터

```typescript
interface TrendData {
  date: YYYY-MM-DD
  sms: { send: number, failed: number }
  rcs: { send: number, failed: number }
  kakao: { send: number, failed: number }
  email: { send: number, failed: number }
  push: { send: number, failed: number }
}
```

### 알림 객체

```typescript
interface DashboardAlert {
  type: 'credential_expiring' | 'quota_alert' | 'pending_approval' | 'inquiry'
  priority: 'high' | 'medium' | 'low'
  message: string
  tenantId?: string
  tenantName?: string
  daysUntilExpiry?: number           // 만료일까지 일 수
  targetPage: string                 // 링크 대상 (예: '/member/company/:id')
  createdAt: ISO8601
}
```

## 검증 · 에러 처리

| 항목 | 검증 | 에러 메시지 |
| --- | --- | --- |
| **기간 선택** | 유효한 범위(1~30일) | (자동 선택, 에러 없음) |
| **테넌트 검색** | 최소 1자 이상 입력 | "사업자번호 또는 회사명을 입력해주세요." |
| **점검 모드** | 사유 ≥10자 필수 | "점검 사유를 10자 이상 입력해주세요." |
| **API 오류** | 백엔드 504/네트워크 | "데이터를 불러올 수 없습니다. 잠시 후 다시 시도해주세요." |

## 위험 액션 · 감사 로그

| 액션 | 위험도 | 감사 | 내용 |
| --- | --- | --- | --- |
| **점검 모드 토글** | 높음 | ✓ 기록 | 토글 전/후 상태 + 사유 + 대상 범위 |
| **카드 클릭/조회** | 낮음 | - | 조회 액션(감사 비대상) |
| **테넌트 검색** | 낮음 | ✓ 기록(선택) | 검색 키워드 + 결과 건수 |
