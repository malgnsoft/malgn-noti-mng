## 1. 개요

고객사 사용자가 **수신자 그룹을 대상으로 스케줄 발송을 계획·실행하는 도메인**. 단발 발송(즉시)과 달리, 캠페인은 여러 채널에 걸쳐 복잡한 시나리오를 지원한다.

**핵심 기능**:
- 캠페인 CRUD (목록/생성/수정/삭제)
- 수신자 그룹 선택 + 채널/메시지 지정
- 발송 시점 설정 (즉시/예약/반복 등)
- 시뮬레이션 (몇 명에게 발송될지 미리보기)
- 테스트 발송 (소수 수신자로 사전 검증)
- 발송 중단/재개
- 복제 (기존 캠페인 템플릿으로 재사용)

## 5. 캠페인 생성/수정 (마법사 또는 단일 페이지)

### 5.1 생성 플로우 (5단계 마법사 추천)

```
Step 1  캠페인 기본정보          →  Step 2  수신자 선택
Step 3  채널 및 메시지          →  Step 4  발송 시점/옵션
Step 5  시뮬레이션 및 컨펌      →  생성 완료 (자동 저장 → 초안)
```

진행률 바: `Step 1 of 5` (각 단계 진행 가능)

### 5.2 Step 1: 캠페인 기본정보

```
┌─────────────────────────────────┐
```

```typescript
interface Campaign {
  id: string
  tenantId: string
  name: string
  description?: string
  
  // 수신자
  recipientGroupId: string
  recipientFilters?: {
    joinedFrom?: ISO8601
    joinedTo?: ISO8601
    purchasedAfter?: ISO8601
    regions?: string[]
    [key: string]: any
  }
  estimatedRecipientCount: number
  
  // 채널 및 메시지
  channels: {
    sms?: { senderId: string, message: string }
    rcs?: { senderId: string, message: string }
    kakao?: { profileId: string, type: 'alimtalk' | 'friendtalk', message: string }
    email?: { domainId: string, subject: string, body: string }
    push?: { message: string, landingUrl?: string }
  }
  
  // 발송 시점
  sendOption: {
    mode: 'now' | 'schedule' | 'repeat'
    scheduleAt?: ISO8601
    repeatRule?: {                  // Cron-like
      frequency: 'daily' | 'weekly' | 'monthly'
      interval?: number
      endDate?: ISO8601
    }
    timeRestriction?: {
      startTime: 'HH:MM'
      endTime: 'HH:MM'
    }
    dailyCap?: number
    concurrency?: number
  }
  
  // 상태
  status: 'draft' | 'scheduled' | 'running' | 'paused' | 'completed' | 'failed' | 'cancelled'
  progress?: {
    sent: number
    success: number
    failed: number
    pending: number
  }
  
  // 예상/실제 비용
  estimatedCost: number
  actualCost?: number
  
  createdAt: ISO8601
  startedAt?: ISO8601
  completedAt?: ISO8601
  updatedAt: ISO8601
}
```

### 캠페인 발송 기록 (CampaignDispatch)

```typescript
interface CampaignDispatch {
  id: string
  campaignId: string
  recipientId: string
  channel: 'sms' | 'rcs' | 'kakao' | 'email' | 'push'
  
  status: 'pending' | 'sending' | 'success' | 'failed'
  failureReason?: string
  
  sentAt?: ISO8601
  deliveredAt?: ISO8601
  createdAt: ISO8601
}
```

## 10. 검증 · 에러 처리

| 항목 | 검증 | 에러 메시지 |
| --- | --- | --- |
| **캠페인명** | 필수, 1~100자 | "캠페인명을 입력해주세요." |
| **수신자 그룹** | 필수, 1명 이상 | "수신자를 선택해주세요." |
| **채널** | 최소 1개 선택 | "최소 1개 채널을 선택해주세요." |
| **메시지** | 채널별 형식 검증 | "메시지를 입력해주세요." |
| **발송 시간** | 과거 시간 불가 (예약 시) | "현재 시간 이후로 선택해주세요." |
| **테스트 수신자** | 이메일/폰번호 형식 | "올바른 형식을 입력해주세요." |
| **API 오류** | 백엔드 504/네트워크 | "작업을 완료할 수 없습니다. 잠시 후 다시 시도해주세요." |
