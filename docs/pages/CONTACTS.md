## 1. 개요

고객사 사용자가 **발송 대상 수신자를 관리하는 도메인**. 개별 연락처(연락처), 수신자 그룹(그룹), 광고/거부 목록(수신거부) 3가지 카테고리로 구성.

**핵심 기능**:
- 연락처 CRUD + 검색 · 필터 · 내보내기
- 그룹 CRUD + 그룹별 연락처 관리
- 수신거부 목록 관리 (휴대폰 · 이메일 · PUSH 토큰)

## 4. 연락처 페이지 (`/contacts/list`)

### 4.1 페이지 구조

```
┌─────────────────────────────────────┐
│ [GNB]                                │  56px
├─────────────────────────────────────┤
│ 크럼: "주소록 · 연락처"              │
│ h1: "연락처"                         │  48px
│ 우측: [+ 추가] [CSV 임포트] [내보내기]│
├─────────────────────────────────────┤
│ 필터 바 (검색/그룹필터/칼럼선택)    │  56px
├─────────────────────────────────────┤
│                                      │
│ [테이블: 연락처 목록]                │  flex-1
│ ┌──────────────────────────────────┐ │
│ │ ☐ 이름 | 휴대폰 | 이메일 | ...   │ │
│ │ ☐ John | 010-... | john@... | │ │
│ └──────────────────────────────────┘ │
│                                      │
│ [페이지네이션] [선택: N개]           │  48px
└─────────────────────────────────────┘
```

### 4.2 테이블 컬럼

| # | 컬럼 | 타입 | 설명 |
| --- | --- | --- | --- |
| 0 | `☐` | 체크박스 | 다중 선택 |
| 1 | 이름 | 텍스트(버튼) | 클릭 시 상세 편집 팝업 |
| 2 | 휴대폰 | 텍스트 | 010-XXXX-XXXX (마스킹 옵션) |
| 3 | 이메일 | 텍스트 | user@example.com |
| 4 | 그룹 | 배지 | 소속 그룹명 (다중 가능) |
| 5 | 생성일 | 텍스트 | YYYY-MM-DD |
| 6 | 액션 | 버튼 | 편집 · 삭제 |

### 4.3 필터 바

#### Quick Chips
```
[전체] [그룹A] [그룹B] [그룹C]
```

#### 검색
```
[검색] ──────────────────────
```
- 범위: 이름 · 휴대폰 · 이메일 부분 일치

#### 기간 필터
```
생성일: [YYYY-MM-DD] ─ [YYYY-MM-DD]
```

#### Advanced
```
─ Advanced ───────────────────
그룹: [다중선택]
채널: ☐ 휴대폰 ☐ 이메일 ☐ PUSH
─────────────────────────────
```

### 4.4 일괄 액션 (체크박스 선택 시)

```
[선택: N개] [그룹 지정] [태그 추가] [CSV] [삭제]
```

- **그룹 지정**: 선택한 연락처를 그룹(들)에 추가
- **태그 추가**: 커스텀 태그 추가 (검색 편의)
- **CSV**: 선택 행 다운로드
- **삭제**: 선택 연락처 일괄 삭제 (확인 팝업)

### 4.5 추가/편집 팝업

#### 추가 (`[+ 추가]` 버튼)

```
┌─────────────────────────────┐
│ 연락처 추가                   │
├─────────────────────────────┤
│ 이름: [text]                │
│ 휴대폰: [010-XXXX-XXXX]    │
│ 이메일: [user@example.com] │
│ 소속 그룹: [체크박스 목록]   │
│                             │
│      [취소]      [저장]    │
└─────────────────────────────┘
```

#### 편집 (행 클릭 또는 액션 버튼)

- 추가 팝업과 동일 구조 + 저장 버튼 텍스트는 "저장"

### 4.6 CSV 임포트

```
[CSV 임포트] → 파일 선택 또는 드래그 앤 드롭
→ 프리뷰 팝업 (첫 5행 표시)
→ "임포트" 버튼 → 진행 토스트 → 완료 토스트
```

## 8. 데이터 모델

### 연락처 (Contact)

```typescript
interface Contact {
  id: string
  tenantId: string
  name: string
  phone: string                       // 010-XXXX-XXXX
  email?: string
  tags?: string[]                     // 커스텀 태그
  groupIds: string[]                  // 소속 그룹 ID 배열
  createdAt: ISO8601
  updatedAt: ISO8601
}
```

### 그룹 (Group)

```typescript
interface Group {
  id: string
  tenantId: string
  name: string
  description?: string
  memberCount: number
  createdAt: ISO8601
  updatedAt: ISO8601
}
```

### 수신거부 (OptOut)

```typescript
interface OptOut {
  id: string
  tenantId: string
  type: 'phone' | 'email' | 'push_token'
  value: string                       // 010-XXXX-XXXX / email@... / Token-...
  reason: string                      // 광고 거부 / 수신거부 / 구독 해지 등
  createdAt: ISO8601
}
```

## 9. 발송 페이지 연동

발송 페이지(`/send/sms` 등)에서 "주소록" 버튼 클릭 시:
1. `AppAddressBookDialog` 팝업 노출 (연락처 또는 그룹 선택 모드)
2. 선택 후 "추가" → 발송 폼의 수신자 목록에 반영
3. 다중 선택 가능 (checkbox)
