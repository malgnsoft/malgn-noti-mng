## 1. 개요

고객사 사용자가 **한 번에 한 채널로 메시지를 발송하는 핵심 도메인**. 5개 채널(SMS/LMS/MMS, RCS, 알림톡/친구톡, Email, Push) 각각의 발송 폼 + 복합(Flow) 조합 발송까지 6개 경로를 제공한다.

**설계 철학**: "한 화면, 한 작업" — 각 채널별 폼은 독립적으로 완결되며, 수신자 선택·첨부·프리뷰·컨펌·초기화의 팝업 풀세트를 공용 컴포넌트로 통일한다.

## 4. 공통 구조 (5+1 채널 동일)

모든 발송 페이지는 다음 레이아웃으로 통일:

```
┌─────────────────────────────────────────┐
│  페이지 헤더                              │ 56px (AppGnb)
├─────────────────────────────────────────┤
│  [탭] SMS · RCS · 알림톡 · 이메일 · PUSH  │ 48px (AppSegmented)
├─────────────────────────────────────────┤
│  본문: 3-칼럼 그리드                      │
│  ┌──────────┬──────────┬──────────┐     │
│  │ 발송 폼  │  프리뷰  │  아이콘  │     │
│  │(col-4)  │ (col-4)  │ (col-4)  │     │
│  └──────────┴──────────┴──────────┘     │
│  팝업: 수신자/템플릿/AI/광고/컨펌/초기화  │
└─────────────────────────────────────────┘
```

### 4.1 탭 네비게이션

- **패턴**: `AppSegmented` (버튼 그룹)
- **위치**: 헤더 아래, sticky (스크롤 시 고정)
- **내용**: SMS · RCS · 알림톡 · 이메일 · PUSH · Flow (6개 탭, 아이콘 + 라벨)
- **동작**: 탭 클릭 시 라우트 변경 (예: `/send/sms` → `/send/rcs`)

### 4.2 본문 레이아웃

**3-칼럼 그리드** (gap-6, 각 칼럼 flex-col):

| 칼럼 | 너비 | 내용 | 스크롤 |
| --- | --- | --- | --- |
| **1. 발송 폼** | 33% | 채널별 입력 폼 (수신자/메시지/옵션) | 개별 스크롤 |
| **2. 프리뷰** | 33% | 채널별 메시지 미리보기 (실시간) | 개별 스크롤 |
| **3. 사이드바** | 33% | 발송 통계/빠른 액션/정보 아이콘 | 개별 스크롤 |

### 4.3 발송 폼 구성 (공통)

#### 섹션 1: 발신 정보 선택

```
┌─ 발신 정보 ────────────────────┐
│ ◎ 발신번호: [Dropdown: 010-xxxx-xxxx] │
│            [추가] [관리]        │
└────────────────────────────────┘
```

- 채널별로 필드 이름만 변경 (발신번호·RCS브랜드·이메일도메인·카카오프로필)
- Dropdown에서 등록된 발신 정보 선택 + `[추가]/[관리]` 버튼 (각각 발신정보 등록 페이지 팝업)

#### 섹션 2: 메시지 입력

**SMS의 예**:

```
┌─ 메시지 ──────────────────────┐
│ 유형:  ◎ SMS  ◉ LMS  ◎ MMS      │
│                               │
│ [제목(LMS/MMS만)]             │
│ [본문 텍스트]                 │
│ ▌ 글자 수: 50/90              │
│                               │
│ [첨부(MMS만)]                 │
└────────────────────────────────┘
```

채널별로 입력 요소가 다름:
- **SMS**: 유형(SMS/LMS/MMS) + 제목(LMS/MMS) + 본문 + 첨부(MMS)
- **RCS**: 제목 + 본문 + 버튼(최대 N개) + 이미지
- **Kakao**: 유형(알림톡/친구톡) + 템플릿 + 메시지
- **Email**: 발신자명 + 제목 + 본문(리치 에디터) + 첨부
- **Push**: 제목 + 본문 + 이미지 + 랜딩 URL

#### 섹션 3: 템플릿 (선택)

```
┌─ 템플릿 ──────────────────────┐
│ [Toggle] 템플릿 사용 안함/사용 │
│ [드롭다운] <템플릿 선택>       │
│ [샘플] [AI 생성] [내 템플릿]   │
└────────────────────────────────┘
```

- 토글로 템플릿 적용 여부 선택
- 적용 시: 메시지 필드는 읽기 전용, 변수 치환만 가능
- 버튼: 샘플 템플릿 팝업 · AI 생성 팝업 · 내 템플릿 관리 팝업

#### 섹션 4: 특수 옵션 (채널별)

**SMS의 예**:

```
┌─ 옵션 ────────────────────────┐
│ 광고성:  ◎ 정보/서비스  ◉ 광고  │
│ 발송시점: ◎ 즉시  ◉ 예약       │
│         [YYYY-MM-DD HH:MM]   │
│                               │
│ 대체문자 설정:  [전개]        │
│ (담당자 연락처 등 자동 첨부)   │
└────────────────────────────────┘
```

다른 채널:
- **RCS**: 예약발송 · BrandKey · 콘텐츠ID 입력
- **Kakao**: 예약발송 · 템플릿 ID 지정 · 마크다운 지원 여부
- **Email**: 트랜잭션/마케팅 선택 · 예약발송
- **Push**: 예약발송 · 배지/사운드/액션 지정

#### 섹션 5: 수신자 입력 (공통)

```
┌─ 수신자 ──────────────────────┐
│ 입력방식: ◎ 직접입력  ◉ 주소록  │
│                               │
│ [Direct] 또는 [주소록 선택]    │
│ 수신자 N명 추가 ✓              │
│ [초기화]                       │
└────────────────────────────────┘
```

- **직접 입력**: 텍스트 영역 (한 줄 1건, 번호/이메일/ID 자동 판별)
- **주소록**: 팝업으로 주소록 선택 (`AppAddressBookDialog`)
- 입력 후 드라이브리스트로 나열 및 삭제 가능
- `[초기화]` 버튼으로 수신자 목록 전체 제거

#### 섹션 6: 발송 버튼

```
┌─────────────────────────────────────┐
│ [취소]            [미리보기] [발송]  │
│                  (확인 팝업 거쳐)     │
└─────────────────────────────────────┘
```

- **취소**: 초기화 확인 팝업 후 폼 리셋
- **미리보기**: (리더기용) 프리뷰 패널 강제 업데이트
- **발송**: 검증 후 `AppSendConfirmDialog` (컨펌 팝업) 표시

## 8. 데이터 모델

### 발송 요청 (통일 구조)

```typescript
interface SendRequest {
  tenantId: string
  channelType: 'sms' | 'rcs' | 'kakao' | 'email' | 'push' | 'flow'
  senderId: string                    // 발신정보 ID (번호/브랜드/도메인/프로필 등)
  
  // 메시지
  subject?: string                    // Email/RCS/Kakao/Push
  body: string
  attachments?: { name: string, data: Buffer }[]
  
  // 수신자
  recipients: {
    type: 'phone' | 'email' | 'userId' | 'deviceToken'
    value: string
  }[]
  
  // 옵션
  sendOption?: {
    mode: 'now' | 'reserve'
    reserveAt?: ISO8601
    adFlag?: boolean              // SMS
    smsType?: 'sms' | 'lms' | 'mms'
    kakaoType?: 'alimtalk' | 'friendtalk'
  }
  
  // 대체문자 (SMS)
  substitutionText?: string
  substitutionVars?: Record<string, string>[]
}
```
